The standard, default atom type. Instances of this class are passed to most of Zedux' hooks and injectors.

An atom defines a template that Zedux will use to create atom instances. Do not confuse "atom instances" with instances of this Atom class! Atom instances are actually instances of [the AtomInstance class](AtomInstance) ðŸ¤•. We refer to instances of this Atom class as simply "atoms".

## Creation

Use [the `atom()` factory](../factories/atom) to create atoms:

```ts
import { Atom, atom } from '@zedux/react'

const exampleAtom = atom('example', 'initial state')

exampleAtom instanceof Atom // true
```

## Extending

When creating your own, custom atom types, you'll usually want to extend this class. Creating your own atom types is an advanced feature and we're not currently documenting it as the internals of these classes may change.

## Methods

<Legend>
  <Item name="getInstanceId">
    <p>Gets a string id for a specific instance of this atom.</p>
    <p>Signature:</p>
    <Ts>{`getInstanceId = (ecosystem, params?) => id`}</Ts>
    <p>
      Accepts an ecosystem and an array of atom params (required if the atom
      takes params, <code>undefined</code> if not). Uses the ecosystem's id
      generator to generate a predictable hash of the atom's key + params combo.
    </p>
    <p>
      Zedux uses this internally to generate atom instance ids. You won't
      typically call this.
    </p>
  </Item>
  <Item name="override">
    <p>Creates a clone of this atom, but with a different value.</p>
    <p>Signature:</p>
    <Ts>{`override = (newValue) => newAtom`}</Ts>
    <p>
      Accepts any of the same{' '}
      <Link to="../factories/atom#value">value types</Link> that{' '}
      <code>atom()</code> accepts. The state, promise, and exports type of the
      new value should match the corresponding types in the overridden atom.
      TypeScript will enforce this.
    </p>
    <p>Returns the new atom</p>
  </Item>

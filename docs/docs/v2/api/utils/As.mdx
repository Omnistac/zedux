A runtime helper for typing [signal events](../classes/Signal#events). This is just a stable no-op function reference that Zedux never calls.

## Examples

Pass different generics to `As` to type custom events:

```ts
interface Todo {
  id: string
  isComplete: boolean
  text: string
}

const todosSignal = injectSignal([] as Todo[], {
  events: {
    addTodo: As<Todo>,
    removeTodo: As<string>,
  },
})

// in a callback or effect:
todosSignal.send('addTodo', { id: '1', isComplete: false, text: 'succeed' })
todosSignal.send('removeTodo', 1) // Type Error! Payload must be a string
```

Usage of `As` is not required. You can either specify no-op functions yourself:

```ts
const todosSignal = injectMappedSignal(
  { todos: todosSignal },
  {
    events: {
      addTodo: () => null as unknown as Todo,
      removeTodo: () => null as unknown as string,
    },
  }
)
```

Or pass TypeScript generics:

```ts
const signal = ecosystem.signal<Todo[], { addTodo: Todo; removeTodo: string }>(
  []
)
```

All Zedux's signal-creating APIs accept all of these methods for typing custom events.

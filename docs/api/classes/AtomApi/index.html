AtomApi

Defines certain one-off properties of an atom. These properties do not fit well in the injector paradigm, as they define key characteristics of the atom itself that should only be set once.

These properties include setting an atom's exports, setting a suspense promise, and setting dispatch and setState interceptors.

All properties added to an AtomApi, except the value, should be stable references. .value is the only property that won't be ignored on subsequent evaluations. It doesn't matter if the properties are not stable; Zedux will just ignore the new references on subsequent iterations.

Creation#

Create AtomApis with the api() factory.

import { api } from '@zedux/react'
const myApi = api()
const withValue = api('some value')
const withStore = api(createStore())
const withExports = api(val).setExports({ ...myExports })
const withPromise = api(val).setPromise(myPromise)
const fromApi = api(myApi)
const addingExports = api(withExports).addExports({ ...moreExports })
const overwritingExports = api(withExports).setExports({ ...newExports })

Usage#

AtomApis can be passed to atom() as the value. They can also be returned from an evaluator function.

import { api, atom, injectStore } from '@zedux/react'
const myAtom = atom('my', api('initial state'))
const withEvaluator = atom('withEvaluator', () => {
return api('initial state')
})
const withStore = atom('withStore', () => {
const store = injectStore('initial state')
return api(store)
})
const withExports = atom('withExports', () => {
const store = injectStore('initial state')
return api(store).setExports({
someProp: 'some val'
})
})
const composingApis = atom('composingApis', () => {
const injectedApi = injectSomethingThatReturnsAnApi()
return api(injectedApi).addExports({
additionalExport: 'some val'
})
})

Properties#

AtomApis expose the following properties:

.dispatchInterceptors#

An array of the dispatch interceptors added to this AtomApi via .addDispatchInterceptor(). May be undefined.

These interceptors function like middleware. They will be called every time instance.dispatch() is called and can cancel the dispatch.

.exports#

The object of the exports added to this AtomApi via .setExports() and/or .addExports(). May be undefined.

.promise#

A promise that Zedux will use to cause React to suspend everywhere this atom instance is used until the promise resolves. Set this via .setPromise().

.setStateInterceptors#

An array of the setState interceptors added to this AtomApi via .addSetStateInterceptor(). May be undefined.

These interceptors function like middleware. They will be called every time instance.setState() is called and can cancel the state setting.

.ttl#

An AtomInstanceTtl or a function that returns an AtomInstanceTtl. If a function, it will be called when Zedux schedules this atom instance's destruction. Will override any ecosystem- or atom-level ttl for this atom instance.

.value#

A reference to the value passed to the api() factory. Can be either a raw value or a Zedux store.

If it's a store and this AtomApi is returned from an evaluator function, the store should be a stable reference that won't change on subsequent evaluations, e.g. by using injectStore().

Methods#

AtomApis expose the following methods:

.addExports#

Accepts an object. The object can contain anything, though all properties should be stable references - memoized functions or ref objects that won't change on subsequent evaluations. Returns the AtomApi for chaining.

Merges the passed object into any already-set exports on this AtomApi. If no exports have been set on this AtomApi, sets the exports.

const initialExports = api(val).addExports({ ...myExports })
const addedExports = api(val)
.setExports({ ...myExports })
.addExports({ ...moreExports })

.addDispatchInterceptor#

Accepts a DispatchInterceptor. Returns the AtomApi for chaining.

Adds the passed dispatch interceptor to this AtomApi's list of dispatch interceptors.

const store = injectStore()
const myApi = api(store).addDispatchInterceptor(
(action, next) => {
const currentState = store.getState()
// make sure you return the state!
if (isBad(currentState)) return currentState // cancel the dispatch
// return the new state
return next(action) // proceed with the dispatch
}
)

.addSetStateInterceptor#

Accepts a SetStateInterceptor. Returns the AtomApi for chaining.

Adds the passed setState interceptor to this AtomApi's list of setState interceptors.

const store = injectStore()
const myApi = api(store).addSetStateInterceptor(
(settable, next) => {
const currentState = store.getState()
// make sure you return the state!
if (isBad(currentState)) return currentState // cancel the state setting
// return the new state
return next(settable) // proceed with the state setting
}
)

.setExports#

Accepts an object. The object can contain anything, though all properties should be stable references - memoized functions or ref objects that won't change on subsequent evaluations. Returns the AtomApi for chaining.

Overwrites any previously-set exports on this AtomApi

const initialExports = api(val).setExports({ ...myExports })
const overwriteExports = api(initialExports).setExports({ ...newExports })

.setPromise#

Accepts a promise. Returns the AtomApi for chaining.

Sets the .promise property of this AtomApi. If this AtomApi is returned from an evaluator, the promise will be set as the atom instance's promise and will be used to cause React to suspend.

This promise should be a stable reference. If you create a new promise on a subsequent reevaluation, Zedux will ignore it.

.setTtl#

Accepts an AtomInstanceTtl or a function that returns an AtomInstanceTtl. If a function, it will be called when Zedux schedules this atom instance's destruction.

Sets the .ttl property of this AtomApi. Overrides any ttl set on this atom or any default ttl on this ecosystem.
